# VSL Playbook — Video Editor Pro

> SOP completo para Video Sales Letter de alta conversão

---

## Estrutura de Conversão (8 Blocos)

```
┌─────────────────────────────────────────────────────────────┐
│ 1. HOOK (0–30s)                                             │
│    → Pattern interrupt + curiosidade/identificação          │
├─────────────────────────────────────────────────────────────┤
│ 2. AUTORIDADE (15–35s)                                      │
│    → Quem fala e por que ouvir                              │
├─────────────────────────────────────────────────────────────┤
│ 3. PROBLEMA (35–70s)                                        │
│    → Dor + custo de não resolver                            │
├─────────────────────────────────────────────────────────────┤
│ 4. MECANISMO (70–140s)                                      │
│    → "Como funciona" + diferencial único                    │
├─────────────────────────────────────────────────────────────┤
│ 5. PROVA (140–220s)                                         │
│    → Resultados, cases, depoimentos, dados                  │
├─────────────────────────────────────────────────────────────┤
│ 6. OBJEÇÕES (220–300s)                                      │
│    → "Mas e se..." + respostas antecipadas                  │
├─────────────────────────────────────────────────────────────┤
│ 7. OFERTA (300–360s)                                        │
│    → O que inclui + valor + garantia                        │
├─────────────────────────────────────────────────────────────┤
│ 8. CTA (final)                                              │
│    → Ação específica + urgência/escassez                    │
└─────────────────────────────────────────────────────────────┘
```

## Regras de Retenção

### Reset Atencional (a cada 20–40s)

| Técnica | Quando Usar | Exemplo |
|---------|-------------|---------|
| **Crop dinâmico** | Fala monótona | Zoom 110% → 100% |
| **Insert de prova** | Após claim forte | Print de resultado |
| **Card de texto** | Número importante | "R$ 50.000/mês" |
| **Mudança de plano** | Transição de bloco | B-roll de produto |
| **Pattern interrupt** | Queda de atenção | SFX + visual abrupto |

### Cortes de Redundância

**Remover sempre:**
- "Então", "bom", "né" (muletas verbais)
- Repetições do mesmo ponto
- Exemplos excessivos (manter 1–2 melhores)
- Transições longas entre blocos

## Inserts Funcionais

### Tipos de Insert

| Tipo | Função | Momento |
|------|--------|---------|
| **Proof Card** | Validar claim | Após promessa |
| **Demo Screen** | Mostrar produto | Bloco mecanismo |
| **Testimonial** | Prova social | Bloco prova |
| **Comparison** | Diferencial | Antes da oferta |
| **Urgency Timer** | Escassez | CTA |

### Especificações Visuais

```
Proof Card:
├── Fundo: brand color ou neutro escuro
├── Borda: 2px branca ou dourada
├── Fonte: Montserrat Bold 48pt (título)
├── Dado: número grande, cor de destaque
├── Fonte: Montserrat Regular 24pt (contexto)
└── Duração: 3–5s
```

## Checklist de Qualidade VSL

### Pré-Edição
- [ ] Roteiro aprovado e final
- [ ] A-roll gravado em boa qualidade
- [ ] B-roll selecionado e organizado
- [ ] Provas coletadas (prints, dados)
- [ ] Brand kit disponível

### Durante Edição
- [ ] Hook nos primeiros 5s
- [ ] Reset atencional a cada 30s
- [ ] Prova inserida após cada claim
- [ ] Legendas em todos os momentos
- [ ] Música abaixo da voz (-12dB)

### Pós-Edição
- [ ] Dead zones eliminadas
- [ ] Áudio limpo (no noise, no hum)
- [ ] Color grading consistente
- [ ] Export em múltiplos formatos
- [ ] Thumbnail forte selecionada
