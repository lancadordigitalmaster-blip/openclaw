## PADRÃO WOLF: WCAG 2.1 NÍVEL AA

```
PRINCÍPIO FUNDAMENTAL:
  Acessibilidade não é feature extra. É parte da qualidade do produto.
  Um produto inacessível está incompleto — não importa como pareça visualmente.

  4 PRINCÍPIOS WCAG (POUR):
  P — Perceivable  (perceptível): conteúdo visível, audível, alternativo
  O — Operable     (operável): tudo funciona via teclado
  U — Understandable (compreensível): linguagem clara, comportamento previsível
  R — Robust       (robusto): funciona com tecnologias assistivas

  Meta Wolf: AA compliant em todos os componentes novos
```

## CHECKLIST COMPLETO DE ACESSIBILIDADE

### 1. Imagens e Mídia

```
  □ Toda <img> tem atributo alt
  □ Imagens decorativas têm alt="" (string vazia — screen reader ignora)
  □ Ícones standalone têm aria-label ou título
  □ Gráficos e charts têm alternativa textual (tabela ou descrição)
  □ Vídeos têm legendas ou transcrição
```

### 2. Formulários

```
  □ Todo input tem <label> associado (htmlFor + id)
  □ Campos obrigatórios marcados com aria-required="true"
  □ Mensagens de erro associadas com aria-describedby
  □ Erros identificados por texto + ícone (não só cor)
  □ Autocomplete configurado em campos comuns (email, name, tel)
  □ Fieldset + legend para grupos de radio/checkbox
```

### 3. Navegação por Teclado

```
  □ Todos os elementos interativos alcançáveis via Tab
  □ Ordem de foco lógica (top-down, left-right)
  □ Focus visível em todos os elementos (não remove outline sem alternativa)
  □ Modais capturam foco (focus trap) e devolvem ao elemento trigger ao fechar
  □ Escape fecha modais e dropdowns
  □ Enter e Space ativam botões e links
  □ Setas navegam em componentes como tabs, menus, selects customizados
  □ Skip link para pular navegação (primeiro elemento da página)
```

### 4. Contraste e Cor

```
  □ Texto normal: contraste mínimo 4.5:1 (verificar com Colour Contrast Checker)
  □ Texto grande (> 18px ou > 14px bold): contraste mínimo 3:1
  □ Elementos de UI (borders, ícones): contraste mínimo 3:1
  □ Informação não transmitida só por cor (ex: erro = vermelho + ícone + texto)
  □ Links distinguíveis do texto por mais de só cor (sublinhado ou negrito)
```

### 5. Estrutura e Semântica

```
  □ Um único <h1> por página
  □ Hierarquia de headings correta (h1 → h2 → h3, sem pular níveis)
  □ HTML semântico: article, section, nav, main, header, footer, aside
  □ Listas usam <ul>/<ol> + <li> (não div/span)
  □ Tabelas têm <th> com scope, <caption> quando necessário
  □ lang="pt-BR" no elemento <html>
```

### 6. Roles e ARIA

```
  □ Usa HTML nativo antes de aria (button em vez de div+role="button")
  □ aria-label quando texto do botão não é suficiente
  □ aria-expanded em botões que abrem/fecham conteúdo
  □ aria-haspopup em botões com dropdown/menu
  □ aria-live="polite" para conteúdo que atualiza dinamicamente
  □ role="alert" para mensagens de erro/sucesso importantes
  □ aria-busy="true" durante carregamento
  □ aria-disabled (não só disabled) para elementos desabilitados customizados
```

## EXEMPLOS: ERROS COMUNS E COMO CORRIGIR

### Formulário com erro

```typescript
// ❌ ERRADO — label não associada, erro sem contexto para screen reader
<div>
  <label>Email</label>
  <input type="email" />
  {error && <span style={{ color: 'red' }}>{error}</span>}
</div>

// ✅ CORRETO — label associada, erro acessível
const EmailField = () => {
  const inputId = 'email-field'
  const errorId = 'email-error'
  const [error, setError] = useState('')

  return (
    <div>
      <label htmlFor={inputId}>
        Email
        <span aria-hidden="true" className="text-red-500 ml-1">*</span>
      </label>
      <input
        id={inputId}
        type="email"
        autoComplete="email"
        aria-required="true"
        aria-invalid={!!error}
        aria-describedby={error ? errorId : undefined}
        className={`border rounded p-2 w-full ${error ? 'border-red-500' : 'border-gray-300'}`}
      />
      {error && (
        <p
          id={errorId}
          role="alert"
          className="text-red-600 text-sm mt-1 flex items-center gap-1"
        >
          <ErrorIcon aria-hidden="true" />
          {error}
        </p>
      )}
    </div>
  )
}
```

### Botão com ícone

```typescript
// ❌ ERRADO — ícone sem nome acessível
<button onClick={handleDelete}>
  <TrashIcon />
</button>

// ✅ CORRETO — aria-label descreve a ação
<button
  type="button"
  onClick={handleDelete}
  aria-label="Excluir campanha Verão 2026"
>
  <TrashIcon aria-hidden="true" />
</button>

// ✅ ALTERNATIVA — texto visível + ícone decorativo
<button type="button" onClick={handleDelete}>
  <TrashIcon aria-hidden="true" />
  <span>Excluir</span>
</button>
```

### Modal com focus trap

```typescript
// ✅ Modal acessível com focus trap e escape
import { useEffect, useRef } from 'react'
import FocusTrap from 'focus-trap-react' // ou implementação manual

interface ModalProps {
  isOpen: boolean
  onClose: () => void
  title: string
  children: React.ReactNode
}

const Modal = ({ isOpen, onClose, title, children }: ModalProps) => {
  const titleId = useId()

  // Fecha com Escape
  useEffect(() => {
    const handleEscape = (e: KeyboardEvent) => {
      if (e.key === 'Escape') onClose()
    }
    if (isOpen) document.addEventListener('keydown', handleEscape)
    return () => document.removeEventListener('keydown', handleEscape)
  }, [isOpen, onClose])

  if (!isOpen) return null

  return (
    // Overlay
    <div
      className="fixed inset-0 z-50 bg-black/50"
      aria-hidden="true"
      onClick={onClose}
    />

    <FocusTrap>
      <div
        role="dialog"
        aria-modal="true"
        aria-labelledby={titleId}
        className="fixed inset-0 z-50 flex items-center justify-center p-4"
      >
        <div
          className="bg-white rounded-lg shadow-xl max-w-lg w-full p-6"
          onClick={e => e.stopPropagation()}
        >
          <div className="flex justify-between items-center mb-4">
            <h2 id={titleId} className="text-xl font-semibold">
              {title}
            </h2>
            <button
              type="button"
              onClick={onClose}
              aria-label="Fechar modal"
              className="p-1 rounded hover:bg-gray-100"
            >
              <XIcon aria-hidden="true" />
            </button>
          </div>

          {children}
        </div>
      </div>
    </FocusTrap>
  )
}
```

### Tabs acessíveis

```typescript
// ✅ Padrão ARIA para tabs
const Tabs = ({ tabs }: { tabs: Tab[] }) => {
  const [activeIndex, setActiveIndex] = useState(0)

  const handleKeyDown = (e: KeyboardEvent, index: number) => {
    if (e.key === 'ArrowRight') setActiveIndex((index + 1) % tabs.length)
    if (e.key === 'ArrowLeft') setActiveIndex((index - 1 + tabs.length) % tabs.length)
    if (e.key === 'Home') setActiveIndex(0)
    if (e.key === 'End') setActiveIndex(tabs.length - 1)
  }

  return (
    <div>
      <div role="tablist" aria-label="Seções de conteúdo">
        {tabs.map((tab, i) => (
          <button
            key={tab.id}
            role="tab"
            aria-selected={i === activeIndex}
            aria-controls={`panel-${tab.id}`}
            id={`tab-${tab.id}`}
            tabIndex={i === activeIndex ? 0 : -1}
            onClick={() => setActiveIndex(i)}
            onKeyDown={e => handleKeyDown(e, i)}
          >
            {tab.label}
          </button>
        ))}
      </div>

      {tabs.map((tab, i) => (
        <div
          key={tab.id}
          role="tabpanel"
          id={`panel-${tab.id}`}
          aria-labelledby={`tab-${tab.id}`}
          hidden={i !== activeIndex}
        >
          {tab.content}
        </div>
      ))}
    </div>
  )
}
```

## CHECKLIST RÁPIDO PRÉ-ENTREGA

```
OBRIGATÓRIO (nunca entrega sem):
  □ Imagens têm alt text
  □ Formulários têm labels associados
  □ Mensagens de erro acessíveis (aria-describedby)
  □ Botões com ícone têm aria-label
  □ Nenhum elemento interativo não alcançável por teclado
  □ Contraste de texto passa 4.5:1 (verifica com ferramenta)
  □ Focus visível em elementos interativos

RECOMENDADO:
  □ Skip link na página
  □ Modais com focus trap
  □ Conteúdo dinâmico com aria-live
  □ HTML semântico (headings, landmarks)
  □ Teste com axe-core sem violações críticas
  □ Teste manual com Tab + Enter + Escape
```
